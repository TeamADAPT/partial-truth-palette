# RFC Template

**Title:** (A brief, descriptive title for the RFC)

**Author:** (Your name)

**Date:** (The date the RFC is created)

**Status:** (Proposed, In Review, Approved, Rejected)

## 1. Summary

A brief, one-paragraph summary of the proposed change.

## 2. Motivation

Why is this change necessary? What problem is it trying to solve?

## 3. Detailed Design

This is the core of the RFC. It should describe the proposed design in detail, including:

*   **Architecture:** A high-level overview of the proposed architecture.
*   **Data Models:** Any changes to the data models.
*   **API Design:** Any changes to the API design.
*   **UI/UX:** Any changes to the user interface or user experience.

## 4. Drawbacks

What are the potential drawbacks of this design? Are there any trade-offs that need to be considered?

## 5. Alternatives

What are the alternatives to this design? Why is this design the best option?

## 6. Unresolved Questions

Are there any unresolved questions that need to be addressed?
